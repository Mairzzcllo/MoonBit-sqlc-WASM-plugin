# ADR-002: Runtime Scope

- **状态**: 已接受
- **日期**: 2026-05-15
- **提出者**: 架构决策

## 背景

sqlc WASM 插件生成的 MoonBit 代码需要引用一组运行时抽象（DB 连接、Row 结果集、Decoder 解码），用于编译和执行生成的查询函数。这些抽象需要定义在 `runtime/` 包中，供生成代码和用户项目导入。

## 决策

### 1. 使用 concrete struct + closure 模式（非 trait）

MoonBit 0.1.20260512 不支持以下特性：
- **Trait 对象**：不能将 trait 用作函数参数类型
- **泛型 trait 方法**：trait 方法不能有类型参数

因此 Runtime 采用 concrete struct + 闭包注入模式：

```moonbit
/// DB handle
pub struct DB {
  exec_fn : (String) -> Int
  execrows_fn : (String) -> Int64
}

/// Row handle
pub struct Row {
  get_fn : (Int) -> String
}
```

构造时注入具体实现闭包，由真实数据库驱动（如 pg driver）提供。

### 2. DB 职责边界

| 方法 | 签名 | 说明 |
|------|------|------|
| `exec` | `(DB, String) -> Int` | 执行 SQL，返回影响行数 |
| `execrows` | `(DB, String) -> Int64` | 执行 SQL，返回 Int64 影响行数 |

**不包含**（延后到后续版本）：
- Transaction (`begin`/`commit`/`rollback`)
- Prepared statement
- Connection pool / lifecycle management
- Query methods with type-safe result decoding

### 3. Row 职责边界

Row 提供按索引访问列值的接口：

```moonbit
pub fn Row::get(self: Row, index: Int) -> String
```

所有数据库类型统一编码为 String，由生成的 struct `decode` 方法按需转换。

### 4. Decoder 为命名约定（非 trait）

生成代码使用 `TypeName::decode(row)` 静态方法约定，而非 Decoder trait。原因：
- MoonBit 0.1 不支持 trait 对象的实例方法作为闭包传递
- 静态方法在调用点更明确，不需要 trait import

生成的 decode 方法签名：
```moonbit
/// Decode a Row into a User struct.
pub fn User::decode(row: Row) -> User {
  { id: row.get(0).to_int(), name: row.get(1) }
}
```

### 5. MVP 边界

**包含**：
- `DB` struct（exec/execrows 闭包）
- `Row` struct（get 闭包）
- 构造函数 `DB::new()`、`Row::new()`

**明确不包含**（延后）：
- Transaction 支持（设计方向：在 DB struct 中增加 begin/commit/rollback 闭包）
- 连接池（由上层驱动管理）
- 类型安全的参数绑定（当前使用 SQL 内 $N 参数）
- 异步/协程支持

## 理由

- **MoonBit 版本限制**: trait 对象和泛型 trait 方法不可用，concrete struct 是唯一可行方案
- **最小运行时**: MVP 只需编译通过，不包含真实 DB 驱动
- **闭包注入**: 灵活解耦，真实驱动可注入任意实现
- **String 编码列值**: 避免泛型复杂性，decode 由生成的类型代码处理

## 后果

- 正向：Runtime 极简，编译快，无外部依赖
- 正向：生成代码可通过常规函数调用与 Runtime 交互
- 负向：`:one`/`:many` 查询的 decode 逻辑在生成代码中为桩（`let _ = db.exec(sql); None`/`[]`），等待 runtime 扩展支持
- 负向：String 编码列值带来运行时类型转换开销

## 替代方案

- **Trait-based abstraction**: MoonBit 0.1 不支持 trait 对象，无法实现
- **Code-generated trait impl**: 仍需 trait 的支持，当前不可行
- **Driver 回调接口**: 复杂性高于 closure 模式，closure 模式更直接
