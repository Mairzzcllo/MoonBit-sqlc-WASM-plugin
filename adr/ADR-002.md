# ADR-002: Runtime Scope

- **状态**: 已接受 (v3 — 2026-05-19 更新)
- **日期**: 2026-05-15 (初版), 2026-05-19 (修订 v2/v3)
- **提出者**: 架构决策

## 背景

sqlc WASM 插件生成的 MoonBit 代码需要引用一组运行时抽象（DB 连接、Row 结果集、Value 参数、错误类型），用于编译和执行生成的查询函数。这些抽象定义在 `runtime/` 包中，供生成代码和用户项目导入。

## 决策

### 1. 使用 concrete struct + closure 模式（非 trait）

MoonBit 0.1.20260512 不支持以下特性：
- **Trait 对象**：不能将 trait 用作函数参数类型
- **泛型 trait 方法**：trait 方法不能有类型参数

因此 Runtime 采用 concrete struct + 闭包注入模式：

```moonbit
pub struct DB {
  exec_fn : (String, Array[Value]) -> Result[Int64, DBError]
  execrows_fn : (String, Array[Value]) -> Result[Int64, DBError]
  query_fn : (String, Array[Value]) -> Result[RowIter, DBError]
  query_row_fn : (String, Array[Value]) -> Result[Row, DBError]
}

pub struct Row { get_fn : (Int) -> String }

pub struct RowIter { next_fn : () -> Result[Option[Row], DBError] }
```

构造时注入具体实现闭包，由真实数据库驱动（如 pg driver）提供。

### 2. DB 职责边界

| 方法 | 签名 | 说明 |
|------|------|------|
| `exec` | `(DB, String, Array[Value]) -> Result[Int64, DBError]` | 执行 SQL，返回影响行数 |
| `execrows` | `(DB, String, Array[Value]) -> Result[Int64, DBError]` | 同 exec，语义不同（sqlc :execrows） |
| `query` | `(DB, String, Array[Value]) -> Result[RowIter, DBError]` | 查询多行（:many） |
| `query_row` | `(DB, String, Array[Value]) -> Result[Row, DBError]` | 查询单行（:one） |

### 3. Value 枚举

参数化查询的类型安全的参数表示：

```moonbit
pub enum Value {
  Null
  Int64(Int64)
  String(String)
  Bool(Bool)
  Double(Double)
  Bytes(Array[Byte])
  Date(Date)
  DateTime(DateTime)
  JsonValue(Json)  // builtin `Json` type from moonbitlang/core/builtin
}
```

### 4. DBError 类型

```moonbit
pub enum DBError {
  ConnectionError(String)
  QueryError(String)
  TypeError(String)
  NoRows
}
```

所有 DB 方法和 Decode 方法统一返回 `Result[T, DBError]`，保留错误传播链。

### 5. Row 职责边界

Row 提供按索引访问列值的类型化 getter 接口：

```moonbit
// 不可空 getter
pub fn Row::get_int64(pos) -> Result[Int64, DBError]
pub fn Row::get_string(pos) -> Result[String, DBError]
pub fn Row::get_bool(pos) -> Result[Bool, DBError]
pub fn Row::get_double(pos) -> Result[Double, DBError]
pub fn Row::get_bytes(pos) -> Result[Array[Byte], DBError]
pub fn Row::get_date(pos) -> Result[Date, DBError]
pub fn Row::get_datetime(pos) -> Result[DateTime, DBError]
pub fn Row::get_json(pos) -> Result[Json, DBError]

// 可空 getter
pub fn Row::get_nullable_int64(pos) -> Result[Option[Int64], DBError]
// ... 其余 7 种类似
```

### 6. RowIter 迭代器

```moonbit
pub fn RowIter::next() -> Result[Option[Row], DBError]
pub fn RowIter::collect() -> Result[Array[Row], DBError]
```

`:many` 查询的迭代载体，支持惰性遍历和一次性收集两种模式。

### 7. Decoder 为命名约定（非 trait）

生成代码使用 `TypeName::decode(row)` 静态方法约定，而非 Decoder trait。原因：
- MoonBit 0.1 不支持 trait 对象的实例方法作为闭包传递
- 静态方法在调用点更明确，不需要 trait import

生成的 decode 方法签名：
```moonbit
pub fn User::decode(row: Row) -> Result[User, DBError] {
  let id = row.get_int64(0)?;
  let name = row.get_string(1)?;
  Ok({ id: id, name: name })
}
```

### 8. MVP 边界更新

**包含（v3）**：
- `DB` struct（含 exec/execrows/query/query_row）
- `Row` struct（含类型化 getter + nullable 变体，空字符串约定为 NULL）
- `RowIter` struct（含 next/collect）
- `Value` 枚举（9 变体：Null/Int64/String/Bool/Double/Bytes/Date/DateTime/JsonValue）
- `Date`/`DateTime` 封装 struct（runtime 内定义，String 包裹）
- `DBError` 枚举（4 变体：ConnectionError/QueryError/TypeError/NoRows）
- 构造函数 `DB::new()`、`Row::new()`、`RowIter::new()`

**规划中**（P1-006）：
- Transaction 支持（Transaction struct + DB::begin）

**明确不包含**：
- 连接池（由上层驱动管理）
- Prepared statement 缓存
- 异步/协程支持

## 理由

- **MoonBit 版本限制**: trait 对象和泛型 trait 方法不可用，concrete struct 是唯一可行方案
- **最小运行时**: MVP 只需编译通过，不包含真实 DB 驱动
- **闭包注入**: 灵活解耦，真实驱动可注入任意实现
- **Value 枚举 + Result 返回**: 类型安全的参数绑定和错误传播，替代旧版 String 编码 + 静默吞错
- **RowIter 模式**: 惰性迭代适配数据流，collect() 提供便捷一次性获取

## 后果

- 正向：Runtime 极简，编译快，无外部依赖
- 正向：生成代码可通过常规函数调用与 Runtime 交互，Result 传播错误
- 正向：类型化 Value 消除 SQL 注入风险（由驱动层参数化执行保障）
- 负向：Row 的 closure 内部需要做类型转换（驱动层返回 String → Value 枚举）
- 负向：RowIter 需要 GC 管理生命周期，流式场景有额外内存分配

## 替代方案

- **Trait-based abstraction**: MoonBit 0.1 不支持 trait 对象，无法实现
- **Code-generated trait impl**: 仍需 trait 的支持，当前不可行
- **Driver 回调接口**: 复杂性高于 closure 模式，closure 模式更直接
- **无 Value 枚举的纯 String 参数**: 放弃类型安全，不可接受
