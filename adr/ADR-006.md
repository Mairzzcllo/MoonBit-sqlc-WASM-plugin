# ADR-006: AST Stability Policy

- **状态**: 已接受
- **日期**: 2026-05-15
- **提出者**: 架构决策

## 背景

Golden Tests 和快照测试需要代码生成器输出具有确定性（deterministic）：同样的输入必须产生完全相同的输出。

## 决策

### 稳定性保证

| 维度 | 策略 | 实现 |
|------|------|------|
| 元素顺序 | Schema 声明序 | `generate_types_from_catalog` 按 schema → table → column 遍历 |
| 别名 | 无依赖 | 仅在查询函数体中使用 `let sql = "..."` 单一绑定 |
| 导入排序 | 不生成 import（下游处理） | 生成代码不含导入语句 |
| 字段排序 | Schema declaration order | `AdaptTable.columns` 保持 protobuf 传入顺序 |
| 格式化 | Emitter 统一控制 | 缩进、换行、空格均由 `Emitter` 负责 |
| 换行符 | LF only | `\n` 硬编码在 Emitter 中 |
| 空白行 | 每个 item 后一个空行 | `emit_source_file` 在 items 间插入 `\n` |
| Doc comment | 保留且格式化 | `///` 前缀，每行独立 `emit_doc_comment` |

### Emitter 契约

Emitter 是纯函数式转换：AST → String，无外部状态依赖。

```moonbit
pub fn emit_source_file(file: SourceFile) -> String
```

- 每次调用创建新的 `Emitter` 实例
- 无全局状态、无缓存、无随机
- 缩进严格使用 2-space

## 理由

- **黄金文件测试的基础**：无确定性则快照测试每次产生不同结果
- **代码审查友好**：同样的语义变更产生同样的 diff
- **增量编译**：文件 hash 不变化则不需要重新编译

## 后果

- 正向：Golden test 可可靠地检测 codegen 变更
- 正向：生成代码格式始终一致
- 负向：Emitter 需要保持向后兼容（不允许随意修改格式）
- 负向：新增 AST 节点类型需同步更新 Emitter

## 替代方案

- **格式化后统一 prettier**：额外依赖链，MoonBit 生态暂无 formatter
- **随机排序 + normalize**：增加复杂性，无实际收益
