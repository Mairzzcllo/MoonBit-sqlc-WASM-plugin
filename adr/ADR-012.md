# ADR-012: Phase D Architecture Gaps — ExecResult, Type Overrides, TimeTZ

- **状态**: 已接受
- **日期**: 2026-05-29
- **提出者**: Phase D 架构评审

## 背景

架构评审对比 MoonBit 插件与 sqlc-gen-go/python/kotlin/typescript，识别 9 个架构差距。本 ADR 记录其中 3 个直接影响运行时和 adapter 层的技术决策，其余 6 个为纯 codegen/配置/测试变更。

## 决策

### 1. GAP-2: ExecResult 完整语义

`:exec` 注解当前返回 `Result[Int64, DBError]`（仅影响行数），缺失 LastInsertId。

变更：runtime 新增 `ExecResult` 结构体，类似 Go 的 `sql.Result`：

```moonbit
pub struct ExecResult {
  last_insert_id : Int64
  rows_affected : Int64
}
```

- `Value` 枚举不新增变体（ExecResult 是 codegen 层构造物，非 wire format 值）
- Codegen `ExecResult + None` 分支由 `-> Result[Int64, DBError]` 改为 `-> Result[ExecResult, DBError]`
- 保留 `db.exec()` 内部签名不变，codegen 层在 DB 方法返回后构造 `ExecResult { last_insert_id, rows_affected }`
- `execrows`/`:execrows` 不变，仍返回 `Result[Int64, DBError]`

### 2. GAP-3: 类型覆盖扩展

当前仅支持 `override_<type>=<moonbit_type>` 格式（P2-002），缺乏 column 级和 nullable 级覆盖。

扩展 adapter 层解析 `GenerateRequest.settings.overrides` 列表：

```yaml
overrides:
  - column: "users.email"
    moonbit_type: "String"
  - nullable: true
    moonbit_type: "Option[Decimal]"
```

覆盖匹配优先级（高→低）：
1. `column` 精确匹配（schema.table.column）
2. `column` 表名+列名匹配（table.column）
3. `column` 仅列名匹配（column）
4. `db_type` + `nullable` 组合匹配
5. `db_type` 单独匹配（即当前 override_<type> 行为）
6. 默认类型映射

### 3. GAP-7: TIMETZ 时区支持

当前 `"time"|"timetz"` 均映射到 `Time { hour, min, sec, micros }`，静默丢弃 PostgreSQL `timetz` 的时区偏移量。

新增独立 struct，不修改现有 `Time`：

```moonbit
pub struct TimeTZ {
  hour : Int
  min : Int
  sec : Int
  micros : Int
  tz_offset : Int  // 时区偏移分钟数，东正西负
}
```

- `Value` 枚举新增 `TimeTZ(TimeTZ)` 变体
- `type_map.mbt` 中 `"timetz"` 由 `Time` 改为 `TimeTZ`
- 新增 `Row::get_timetz(pos)` / `Row::get_nullable_timetz(pos)` 一对 getter
- `type_to_value_constructor` 新增 `"timetz" → TimeTZ` 分支
- 向后兼容：所有现有 `"time"` 映射不变

## 理由

- **ExecResult 结构体**: 与 Go `sql.Result` 对齐，满足 ORM/DAO 层获取自增 ID 的核心需求。独立结构体而非元组提升可读性。
- **Overrides 三层级**: 与 sqlc-gen-go 的 override 配置完全兼容，降低用户迁移成本。`column` 精确匹配处理 schema 限定名（`public.users.email`）消除歧义。
- **TimeTZ 独立 struct**: 保持向后兼容（现有 Time 用户不受影响）。`tz_offset` 用分钟而非秒，对齐 PostgreSQL `timezone_offset` 返回分钟单位的惯例。

## 后果

- 正向：ExecResult 使自动递增主键场景可用
- 正向：Type override 配置格式与 sqlc-gen-go 一致，降低学习成本
- 正向：TimeTZ 零成本抽象，不影响现有 Time 用户
- 负向：ExecResult codegen 路径需区分 ExecResult+Rows（走 decode）和 ExecResult+None（走 exec+构造结构体）
- 影响：Golden 测试需更新以反映 ExecResult 结构体输出和 TimeTZ 类型映射
