# ADR-004: Naming Convention — snake_case → PascalCase

- **状态**: 已接受
- **日期**: 2026-05-15
- **提出者**: 架构决策

## 背景

生成代码需要一致的命名规范。数据库对象（表、列、枚举值）通常使用 snake_case，MoonBit 代码使用 PascalCase（类型）和 snake_case（函数/字段）。

## 决策

### 命名映射表

| 源 (SQL) | 目标 (MoonBit) | 示例 |
|---------|---------------|------|
| 表名 | PascalCase struct | `users` → `Users` |
| 列名 | snake_case field | `created_at` → `created_at` |
| 枚举名 | PascalCase enum | `user_role` → `UserRole` |
| 枚举值 | PascalCase variant | `admin` → `Admin` |
| 查询函数 | `query_<name>` | `list_users` → `query_list_users` |
| 结果行类型 | `<QueryName>Row` | `get_user` → `GetUserRow` |

### 函数签名规范

每个查询函数以 `db: DB` 作为第一参数，后跟类型映射后的查询参数：

```moonbit
pub fn query_get_user(db: DB, id: Int) -> Option[GetUserRow]
pub fn query_list_users(db: DB) -> Array[ListUsersRow]
pub fn query_delete_user(db: DB, id: Int) -> Int
```

### 实现

`snake_to_pascal(s)` 将 snake_case 拆分后每段首字母大写：

```moonbit
"user_profiles" → "UserProfiles"
"blog_post_tags" → "BlogPostTags"
```

## 理由

- **MoonBit 社区惯例**：遵循标准 MoonBit 命名风格
- **避免歧义**：PascalCase 类型与 snake_case 函数区分清晰
- **确定性**：snake_case → PascalCase 转换是纯算法，无歧义

## 后果

- 正向：生成代码风格一致，符合 MoonBit 社区习惯
- 正向：查询函数名自带 `query_` 前缀，便于 IDE 搜索过滤
- 负向：少数边缘 case（如全大写缩写）需要特殊处理（可后续扩展）

## 替代方案

- **保持 snake_case 原名**：不符合 MoonBit 类型命名规范
- **camelCase**：MoonBit 社区以 snake_case 为主流
