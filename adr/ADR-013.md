# ADR-013: 100 Edge Cases Analysis — Classification and Fix Feasibility

- **状态**: 已接受
- **日期**: 2026-05-30
- **提出者**: 100 边界情况系统分析（2026-05-30）

## 背景

在对 MoonBit sqlc WASM 插件进行 100 个边界情况系统分析后，发现约 50% 已被历史任务覆盖或为设计决策，其余约 50 个需要修复。本 ADR 记录分类标准、设计决策保留列表，以及新任务的任务化映射。

## 分类方法论

每个边界情况按以下 4 个维度评估：

1. **风险等级**: P0（数据损坏/崩溃）| P1（功能缺口）| P2（质量改进）
2. **是否已修复**: 检查历史 Phase 任务是否已覆盖
3. **是否设计决策**: 是否为 MVP 边界或意愿设计
4. **修复可行性**: 简单/中等/困难

### 设计决策保留 (20 项)

以下边界情况视为可接受风险或意愿设计：

1. **#14 未知 PG 类型→String**: 静默回退是最小惊讶原则
2. **#16 money→Double**: money 极少使用，String fallback 可接受
3. **#17-19 无格式验证**: wrapper 类型保持轻量，验证留到应用层
4. **#22 数组可空性混淆**: 由 sqlc 元数据保证
5. **#26 未知 cmd→Exec**: 安全默认，等价于没有 RETURNING 的 INSERT
6. **#29/#30 无参数/空查询**: 合法边界，codegen 正常处理
7. **#34 field.not_null 信任 sqlc**: sqlc 是源可信元数据
8. **#38-40 null_mask 静默安全**: 超索引视为非 null 是合理默认
9. **#44 wrapper String 无验证**: 保持构造器轻量
10. **#45 数组类型不全**: JSON 数组解码已覆盖大部分场景
11. **#48 MoonBit Int→Int64**: 语言设计
12. **#61 0 字段 struct**: 生成的 Ok({}) 合法
13. **#64 begin_fn 总是 Err**: 依赖驱动实现
14. **#71 collect 在 Err 停止**: RowIter 流式语义
15. **#85 NoRows 上下文**: 已是最详尽的错误变体
16. **#87 重复 import**: 实际只有一个 import
17. **#91-96 ADR 限制**: 已在 ADR-009 中记录

```
所有 P1 独立于 P0，可并行执行
```

## 理由

- **P0-063 评为困难**: 字段按列名解码需要改动 Row 结构体（增加列名映射）和 decode 代码生成路径，是架构级变更，但影响仅限于正确性场景
- **P0-061 合并 4 个越界问题**: 都在 codec.mbt 同一文件中，可以一次修复
- **不修复设计决策**: 20 项设计决策保留均有明确的合理性论证，避免过度工程化
- **P0-067 评为简单**: 只需在启动时增加一个指针范围断言

## 后果

- 正向：100 边界情况全部有明确处理路径（修复/设计决策/已知已修复）
- 正向：7 + 10 = 17 个新任务与现有 9 个 Phase D 任务无冲突
- 正向：P0-062/063/065/066 从源头消除数据损坏和编译失败风险
- 负向：Sprint 范围显著扩大（由 9 任务增至 26 任务）
- 负向：P0-063 字段列名解码是架构级变更，需谨慎设计
