# ADR-014: Plugin Options Extension (GAP-4)

- **状态**: 已接受
- **日期**: 2026-05-30
- **提出者**: Phase D 架构差距消除 Sprint
- **关联任务**: P0-057

## 背景

sqlc WASM 插件通过 `plugin_options` 配置项（位于 `sqlc.yaml` 的 `codegen:` 段）向生成器传递配置参数。在 Phase D 评审前，插件仅支持 3 个选项：

1. `package_name` / `package` — 输出包名
2. `emit_json_tags` — 生成 `@json.tag()` 注解
3. `emit_db_tags` — 生成 `@db.tag()` 注解

相比之下，成熟的 sqlc 插件（如 `sqlc-gen-go`）支持 15+ 配置选项。GAP-4 要求将插件选项数量扩展到与社区生态匹配的水平。

## 决策

### 新增 8 个配置选项

在 `parse_plugin_options` 中增加以下选项的解析，并通过 `PluginOptions` 结构体传递到 codegen 管道：

| 选项 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `emit_sql_as_comment` | bool | true | 在生成函数上方以 doc comment 形式嵌入 SQL 原文 |
| `omit_unused_structs` | bool | false | 跳过未被任何查询引用的 struct 类型生成 |
| `emit_empty_slices` | bool | false | :many 查询无结果时返回空数组而非 Err(NoRows) |
| `initialisms` | string | "" | 逗号分隔的初始isms字典（如 "API,HTTP,ID"） |
| `json_tags_case_style` | string | "snake" | JSON 标签命名风格：snake / camel / pascal |
| `emit_exact_table_names` | bool | false | 使用表名原样作为 struct 名，不转换单复数 |
| `emit_methods_with_db_argument` | bool | false | 生成附加方法变体，显式接收 db 参数（实验性） |

### 架构变更

1. **新建 `plugin/naming.mbt`** — 独立模块处理命名转换逻辑：
   - `initialisms` 字典管理（用户自定义 + 内置常见 initialisms）
   - `json_tags_case_style` 风格转换器（snake_case / camelCase / PascalCase）
   - 隔离命名逻辑使 emitter 和 type_codegen 职责清晰

2. **`PluginOptions` 结构体扩展** — 新增字段对应 8 个选项，`parse_plugin_options` 解析键值对字符串

3. **`emit_empty_slices` 行为决策** — 在 `:many` 查询中，当 `db.query` 返回 `Err(NoRows)` 且 `emit_empty_slices=true` 时，codegen 生成 `match result { Err(NoRows) => Ok([]), _ => result.map(fn(rows) { ... }) }` 模式，将无结果转换为空数组。

4. **`query_parameter_limit` 已移除 (P1-053)** — 原计划在此任务中实现，但 codegen 管道在参数化阶段前即完成查询函数生成，参数限制需要 SQL 级查询拆分（非 codegen 插件职责）。已在 P1-053 中彻底移除。

### 向后兼容

所有新增选项的默认值均保持现有行为（false/空/零），现有 `sqlc.yaml` 配置无需任何修改。

## 备选方案

1. **不扩展选项** — 简单但导致插件竞争力不足，拒绝了
2. **JSON 格式选项** — 用户编写复杂的 JSON 结构增加心智负担，拒绝了
3. **go-style 配置结构** — sqlc-gen-go 使用嵌套 `overrides`/`rename` 结构，但因 sqlc WASM 插件 `plugin_options` 传参限制，平铺 `key=value` 格式更简单
4. **`emit_empty_slices` 在适配器层过滤** — 尝试在 RowIter 层处理，但会导致与 Runtime 协议耦合过紧，最终选定 codegen 层转换

## 影响范围

- **plugin/adapter.mbt**: `parse_plugin_options` 函数扩展，`PluginOptions` 结构体新增字段
- **plugin/naming.mbt**: 新建文件，initialisms + case style 逻辑
- **plugin/main.mbt**: 适配器层 thread options 到 codegen 管道
- **plugin/emitter.mbt**: `emit_sql_as_comment` 影响 emitter 输出
- **plugin/type_codegen.mbt**: `omit_unused_structs` 控制类型生成范围
- **plugin/query_codegen.mbt**: `emit_empty_slices` 影响查询函数 body 生成

## 测试

- 555 个 inline 测试全部通过
- 覆盖每个选项的独立解析测试
- Golden 测试验证 `emit_sql_as_comment`、`omit_unused_structs`、`emit_empty_slices` 的输出差异
- 默认值向后兼容性测试

## 后果

### 正向

- 插件功能完整性显著提升，接近社区插件水平
- 所有新选项均向后兼容
- `naming.mbt` 隔离命名逻辑，降低了 emitter 和 type_codegen 的耦合
- `emit_empty_slices` 在 codegen 层处理而非 runtime 层，保持 runtime 通用性

### 负向

- 选项解析仍使用平铺 `key=value` 格式，不支持嵌套结构
- 新选项（特别是 `emit_exact_table_names`）可能与现有命名约定产生交互，需要更多集成测试验证
