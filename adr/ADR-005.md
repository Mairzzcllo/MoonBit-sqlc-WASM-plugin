# ADR-005: Type Mapping Policy

- **状态**: 已接受
- **日期**: 2026-05-15
- **提出者**: 架构决策

## 背景

sqlc 解析 SQL 查询后报告 PostgreSQL 列类型（int4、text、timestamptz 等）。Codegen 需要将这些类型映射到 MoonBit 类型。

## 决策

### 映射表

| PG 类型 | MoonBit 类型 | 备注 |
|---------|-------------|------|
| `int2`, `int4`, `smallint`, `integer`, `int` | `Int` | |
| `int8`, `bigint` | `Int64` | |
| `float4`, `real` | `Float` | |
| `float8`, `double`, `numeric`, `decimal` | `Double` | |
| `bool`, `boolean` | `Bool` | |
| `text`, `varchar`, `char`, `bpchar`, `name` | `String` | |
| `bytea` | `Bytes` | |
| `json`, `jsonb` | `JsonValue` | |
| `uuid`, `inet`, `cidr`, `macaddr` | `String` | MVP fallback |
| `date`, `time`, `timetz`, `timestamp`, `timestamptz` | `String` | MVP fallback |
| `interval` | `String` | MVP fallback |
| 未知类型 | `String` | 安全 fallback |

### 数组类型

PG 数组通过 `Arr(inner)` 包装，映射为 `Array[T]`：

```moonbit
Arr(Named("int4")) → Array[Int]
Arr(Arr(Named("text"))) → Array[Array[String]]
```

### Nullable 包装

参见 ADR-003：可为空列映射为 `Option[T]`。

## 理由

- **安全 fallback**：未知类型映射为 `String` 保证生成代码始终可编译
- **渐进增强**：映射表可在 ADR 更新时扩展，无需修改 codegen 逻辑
- **数组递归**：通过 IR 的 `Arr` 类型自然处理多维数组

## 后果

- 正向：映射表集中管理，修改映射只需改 `type_map.mbt`
- 正向：未知类型不导致编译失败
- 负向：时间/日期类型使用 String 不保留时区语义（后续可扩展为专用类型）
- 负向：numeric/decimal 映射为 Double 可能丢失精度

## 替代方案

- **编译期报错**：未知类型导致失败，过于严格
- **自定义类型映射**：允许用户配置映射文件，超出 MVP 范围
