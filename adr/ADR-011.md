# ADR-011: Codegen Build Body — result_shape Priority and Method Dispatch

- **状态**: 已接受
- **日期**: 2026-05-26
- **提出者**: Phase C (P1-026, P1-029, P2-008)

## 背景

sqlc 插件 codegen 层的 `build_body` 函数负责将 IR 查询转换为 MoonBit 函数体。此前存在三个设计缺陷：

1. **RETURNING * 被忽略**: `ExecResult`/`ExecCount` 分支无视 `query.result_shape`，始终走 `conn.exec()` 返回 `Int64`。但 PostgreSQL 的 `DELETE/RETURNING *` 或 `UPDATE/RETURNING *` 有结果列，应像 `:one`/`:many` 一样解码。
2. **raw_cmd 不区分**: `ExecResult` 分支在所有 `raw_cmd`（Exec/CopyFrom/Batch/ExecLastId）下都调用 `.exec()`，`runtime/db.mbt` 已实现的 `.copyfrom()`/`.batch()`/`.execlastid()` 从未被使用。
3. **Transaction 过度生成**: `generate_query_fns` 为每个查询生成 `db: DB` 和 `tx: Transaction` 两个重载，但 `Transaction` 结构体没有 `copyfrom`/`batch`/`execlastid` 方法。

## 决策

### 1. result_shape 优先于 raw_cmd

`build_body` 按照以下优先级判断代码生成路径：

```
result_shape → Rows(fields) → decode 路径（无视 raw_cmd）
             → None         → raw_cmd dispatch 路径
```

- `ExecResult + Rows(fields)` → `conn.query_row() + T::decode()`（单行解码）
- `ExecCount + Rows(fields)` → `conn.query() + iter.collect() + for 循环 decode`（多行解码）
- `ExecResult + None` → 按 raw_cmd 分发
- `ExecCount + None` → `conn.execrows()`（不受 raw_cmd 影响）

### 2. raw_cmd 精确分发

对于 `ExecResult + None` 路径，按 `query.raw_cmd` 分发到 DB 的具体方法：

| raw_cmd | 生成调用 |
|---------|---------|
| `CopyFrom` | `db.copyfrom(sql, params)` |
| `Batch` | `db.batch(sql, params)` |
| `ExecLastId` | `db.execlastid(sql, params)` |
| `Exec` (及其他) | `db.exec(sql, params)` |

### 3. Transaction 重载条件生成

`generate_query_fns` 通过 `supports_transaction(cmd: QueryCmd) -> Bool` 判定：

| raw_cmd | 生成 tx 重载? | 原因 |
|---------|--------------|------|
| `One` | ✅ Yes | Transaction 有 `query_row` |
| `Many` | ✅ Yes | Transaction 有 `query` |
| `Exec` | ✅ Yes | Transaction 有 `exec` |
| `ExecRows` | ✅ Yes | Transaction 有 `execrows` |
| `CopyFrom` | ❌ No | Transaction 无 `copyfrom` |
| `Batch` | ❌ No | Transaction 无 `batch` |
| `ExecLastId` | ❌ No | Transaction 无 `execlastid` |

## 理由

- **正确性**: RETURNING * 语义是 sqlc 核心功能，忽略 `result_shape` 导致生成不可用的代码（编译通过但运行时语义错误）
- **完整性**: runtime/db.mbt 已实现的方法应在 codegen 中被正确调用
- **最小化**: 避免生成调用不存在方法的重载（编译错误）或不必要的代码膨胀
- **渐进性**: 语义完整后，后续 Phase C 的 emit_interface、type_override 等特性可以在此基础上构建

## 后果

- 正向：RETURNING * 查询现在生成类型安全的解码代码
- 正向：runtime/db.mbt 的 copyfrom/batch/execlastid 方法被正式接入 codegen
- 正向：Transaction 重载减少 ~40%（跳过 3/7 类型），生成代码体积缩小
- 负向：用户若自定义 `raw_cmd` 映射，需同步更新 `build_body` 和 `supports_transaction`
- 影响：Golden 测试需更新以反映 ExecResult+Rows 的新输出路径
