# ADR-009: Known Limitations and MVP Boundaries

- **状态**: 修订 v2
- **日期**: 2026-05-20
- **提出者**: P0-039 文档补全 / P0-044 兼容性分析

> v2 修订: P0-044 发现 wazero 可加载含 GC 类型注解的 WASM 二进制，
> 更新第 5 节入口点架构和新增第 6 节帧头协议限制。

## 背景

在 P0 阶段，插件实现了 MVP（最小可行产品）功能集。部分类型映射、查询特性和编解码策略采用了简化方案，存在已知的精度损失和功能限制。本文档集中记录这些限制，作为后续迭代的参考依据。

## 已知限制

### 1. 类型映射精度损失

| PostgreSQL 类型 | 映射目标 | 限制说明 |
|----------------|----------|---------|
| `numeric` / `decimal` | `Double` | PostgreSQL numeric(1000,0) 支持 ~1000 位有效数字，Double 仅有 15-17 位精度。后续应映射为 String 或自定义 Decimal 类型 |
| `interval` | `String` | MVP 占位符，不提供结构化访问。后续应映射为 Duration 或 Int64（微秒数） |
| `uuid` | `String` | 无 UUID 类型校验，纯字符串传递 |
| `inet` / `cidr` / `macaddr` | `String` | 网络类型用 String 传递，失去结构化校验 |
| `date` / `time` / `timetz` | `String` | 时间类型按字符串传递，无时区或格式校验 |
| `timestamp` / `timestamptz` | `String` | MVP 占位符，后续应映射为专用时间类型 |

### 2. RETURNING * MVP 限制

- `RETURNING *` 子句产生的列被 IR 层识别为 `ExecResult` 或 `ExecCount` 查询的结果列（P0-035 修复）。
- 当前行为：当 ExecResult/ExecCount 有返回列时，`build_result_shape` 返回 `Rows(fields)`。
- 限制：返回列的类型推导完全依赖 sqlc 提供的元数据，插件不做独立的 SQL 解析来验证列类型正确性。

### 3. 编解码安全边界

代码层（codec.mbt）采用以下安全策略：

| 防御机制 | 实现 | 说明 |
|---------|------|------|
| Varint 最大 5 字节 | `Encoder::write_varint` 上限 5 字节，`Decoder::read_varint` 5 次迭代 | 防止恶意超大 varint 导致的无限循环（P0-033 修复） |
| 掩码保护 | `val & 0x7F` 确保编码只输出低 7 位 | 防止算术右移在 wasm32 上的符号扩展 |
| read_string 边界钳制 | `actual_len = min(len, available)` | 防止长度字段超出缓冲区导致的越界读取 |
| decode_embedded 边界 | 验证 `len <= remaining()`，超限时返回空子解码器 | 防止嵌套消息长度欺骗导致的缓冲区越界 |
| skip_field 未知类型 | 未知 wire type 跳转到消息末尾 | 保持与未来 proto 扩展的兼容性 |

### 4. Array 类型解码限制

- 当前 `map_internal_type` 将 `TEXT[]` 映射为 `Array[String]`（通过 `Arr` 递归包装）。
- 生成代码的 `decode` 函数对数组列的处理与标量列相同：`Some(row.get(N))`。
- 限制：Row::get 返回字符串；数组列的 protobuf 编码在 sqlc 层面已是逗号分隔的字符串。无专有的 JSON/Protobuf 数组解码逻辑。

### 5. WASM 平台兼容性

#### 5.1 wazero GC 类型兼容性

- MoonBit `--target wasm` 输出包含 GC 类型注解（refany 类型签名，非标准 WASM MVP）。
- **实测确认**（P0-044，2026-05-20）：sqlc v1.31.1 使用的 wazero 运行时**可以加载和执行**含 269 个 refany 类型注解的 WASM 二进制（270KB）。
- 早期文档中 "wazero 不支持 GC 类型" 的描述不准确，已修正。

#### 5.2 入口点架构

`_start` → `__moonbit__main` → `run_io_loop()`（验证：wasm2wat 确认此调用链，P0-038 #13）。`run_io_loop` 在 moonrun 下不执行（无 WASI），仅在 wazero（sqlc generate）下触发。

### 6. I/O 协议不匹配（当前主要阻塞）

- sqlc WASM 插件协议要求**无帧格式的原始 stdin/stdout protobuf**（参考 sqlc-gen-greeter）。
- 当前 wasi_io.mbt 使用 4 字节 LE 帧头协议（read_frame_header + write_frame）。
- 此不匹配导致 `sqlc generate` 报错 `proto: cannot parse invalid wire-format data`。
- 修复任务：P0-045，将 run_io_loop 改为 `read_all(fd)` + `fd_write` 直接模式。

1. **P1**: 为 `uuid` / `inet` 添加校验包装类型
2. **P1**: 将 `numeric` 映射为 String 或提供 BigDecimal runtime 类型
3. **P2**: 为 `interval` 添加 Duration 映射
4. **P2**: 完善 Array 类型解码的专用逻辑
5. **P2**: 添加 JSON/Protobuf 级别列类型校验
