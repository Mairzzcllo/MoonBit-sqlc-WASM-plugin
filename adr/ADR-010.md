# ADR-010: Transaction Support — Concrete Struct + Closure Pattern

- **状态**: 已接受
- **日期**: 2026-05-19
- **提出者**: P1-006

## 背景

Runtime 需要支持数据库事务操作，允许用户在单个事务上下文中执行多条 SQL 并在最后提交或回滚。此前 runtime 仅提供无状态 `DB` 结构体，没有事务抽象。

## 决策

采用与 `DB` 相同的 **concrete struct + closure** 模式实现 `Transaction`：

```moonbit
pub struct Transaction {
  exec_fn : (String, Array[Value]) -> Result[Int64, DBError]
  execrows_fn : (String, Array[Value]) -> Result[Int64, DBError]
  query_fn : (String, Array[Value]) -> Result[RowIter, DBError]
  query_row_fn : (String, Array[Value]) -> Result[Row, DBError]
  commit_fn : () -> Result[Unit, DBError]
  rollback_fn : () -> Result[Unit, DBError]
}
```

关键设计点：
- `Transaction` 不继承 `DB`，不引入 trait 或接口（MoonBit 0.1 不支持 trait 对象和泛型 trait 方法）
- 事务的 `exec`/`execrows`/`query`/`query_row` 签名与 `DB` 完全一致，方便 codegen 复用
- `commit` 和 `rollback` 返回 `Result[Unit, DBError]`，允许驱动返回错误
- `DB` 新增 `begin_fn` 字段和 `DB::begin()` 方法，委派给驱动实现

## 理由

- **一致性**: 与 `DB`/`Row`/`RowIter` 使用相同的 concrete struct + closure 模式，不引入新抽象
- **灵活性**: 驱动可以自定义 begin/commit/rollback 行为，甚至返回错误表示不支持事务
- **渐进式**: 现有 `DB` 代码无需修改，新增 `begin_fn` 是向后兼容的扩展（构造时需要提供默认实现）
- **最小 runtime**: Transaction 结构体只包含 closures，无状态存储，无额外内存开销

## 后果

- 正向：codegen 可实现 `with_transaction` 辅助函数或生成事务重载
- 正向：MockDB 可模拟事务成功/失败场景
- 负向：`DB::new` 签名需要额外 `begin_fn` 参数，所有调用点（包括测试）需更新
- 负向：integration test stubs 需要添加 `Transaction` 和 `begin_fn` 的镜像定义
